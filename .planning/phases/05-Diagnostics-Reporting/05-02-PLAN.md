# Plan 05-02: Diagnostic Logging Enhancement

## Objective

Add comprehensive diagnostic logging to the thermal simulation engine to enable detailed debugging of validation failures. Diagnostics should include:
- Hourly zone temperatures (all zones)
- Hourly thermal mass temperatures
- Hourly surface temperatures
- Hourly load breakdown: solar gains, internal gains, HVAC, inter-zone transfer
- Energy accumulation tracking
- Sensitivity and free-floating temperatures

This addresses requirement REPORT-02.

## Background

When validation fails, it's currently difficult to understand *why* without adding ad-hoc `println!` statements to the simulation code. Systematic, structured diagnostics will accelerate debugging of validation issues like Case 900 cooling under-prediction and Case 960 inter-zone effects.

## Locked Decisions

1. **Logging Framework:** Use Rust's `log` crate with `env_logger` backend. Standard, flexible, no custom implementation.
2. **Verbosity Control:** `RUST_LOG` environment variable controls output level:
   - `info`: Summary statistics only (low overhead)
   - `debug`: Hourly values printed to stderr (moderate overhead)
   - `trace`: Full per-timestep details (high overhead, for deep debugging)
3. **Output Destination:** stderr (not stdout) to keep test output clean
4. **Data Collection:** Collect into in-memory structures during simulation, export at end (don't write per-timestep unless trace mode)
5. **CSV Export:** Use `csv` crate for easy pandas/numpy analysis in Python

## Tasks

### Task 1: Create Diagnostics Module

**Priority:** P0
**Estimate:** 1 hour

Create `src/validation/diagnostics.rs` with structures:

```rust
/// Collected diagnostic data for a single simulation run
pub struct SimulationDiagnostics {
    /// Timestamps (hour indices)
    pub hours: Vec<usize>,
    /// Zone temperatures (°C) - indexed by [timestep][zone]
    pub zone_temps: Vec<Vec<f64>>,
    /// Mass temperatures (°C)
    pub mass_temps: Vec<Vec<f64>>,
    /// Surface temperatures (°C)
    pub surface_temps: Vec<Vec<f64>>,
    /// Load breakdown per timestep (Watts)
    pub loads: LoadBreakdown,
    /// Cumulative energy (kWh)
    pub cumulative_energy: EnergyAccumulation,
}

/// Breakdown of thermal loads at each timestep
pub struct LoadBreakdown {
    pub solar: Vec<Vec<f64>>,      // Solar gains per zone
    pub internal: Vec<Vec<f64>>,   // Internal gains per zone
    pub hvac: Vec<Vec<f64>>,       // HVAC output per zone (positive=heating)
    pub inter_zone: Vec<Vec<f64>>, // Inter-zone transfer per zone
    pub infiltration: Vec<Vec<f64>>, // Infiltration losses per zone
}

/// Energy accumulation over simulation
pub struct EnergyAccumulation {
    pub heating_kwh: Vec<f64>,   // Cumulative heating by zone
    pub cooling_kwh: Vec<f64>,   // Cumulative cooling by zone
    pub total_kwh: Vec<f64>,     // Total energy by zone
}
```

Implement:
- `SimulationDiagnostics::new(num_zones: usize, num_timesteps: usize) -> Self`
- `SimulationDiagnostics::record_timestep(&mut self, hour: usize, model: &ThermalModel)`
- `SimulationDiagnostics::export_csv(&self, path: &Path) -> Result<()>`
- `SimulationDiagnostics::print_summary(&self)`

**Acceptance:**
- Structs compile and store data correctly
- Memory footprint acceptable (~10KB per zone per 8760 hours)

### Task 3: Integrate with Validation

**Priority:** P1
**Estimate:** 20 minutes

Modify `src/validation/ashrae_140_validator.rs` to optionally collect diagnostics:

```rust
pub fn validate_case_with_diagnostics(
    case: ASHRAE140Case,
    collect_diags: bool,
) -> (ValidationReport, Option<SimulationDiagnostics>) {
    if collect_diags {
        let mut diags = SimulationDiagnostics::new(...);
        model.set_diagnostics(Some(diags));
        // ... run simulation
        let diag = model.get_diagnostics();
        (report, diag)
    } else {
        // ... run without diagnostics
        (report, None)
    }
}
```

**Acceptance:**
- Validation tests can opt-in to diagnostics
- Diagnostics returned match simulation state

# Plan 05-01: Validation Report Generation

## Objective

Implement automated Markdown report generation for ASHRAE 140 validation results. The report should provide:
- Overall pass/fail status with pass rate percentage
- Per-case energy and peak load metrics with reference ranges
- Color-coded pass/fail indicators (green/red)
- Summary of systematic issues across all cases
- Recommendations for next steps

## Background

Currently, validation results are scattered across test outputs. There's no consolidated report that summarizes all case results in a format suitable for tracking progress toward 100% validation compliance. This plan addresses requirement REPORT-01.

## Locked Decisions

1. **Format:** Markdown (not HTML or PDF) for GitHub compatibility and easy editing
2. **Trigger:** Report generation on demand via explicit command (not automatic after every test)
3. **Location:** Reports saved to `docs/ASHRAE140_RESULTS.md` (version-controlled)
4. **Content:** Include both summary and detailed case-by-case tables
5. **Styling:** Use simple tables with emoji indicators (✅/❌) for pass/fail

## Tasks

### Task 1: Design Report Structure

**Priority:** P0 (Critical)
**Estimate:** 30 minutes

Design the Markdown report structure with the following sections:

1. **Header** - Project name, generation timestamp, overall statistics
2. **Summary Card** - Pass rate, MAE, max deviation, cases passing/failing
3. **Detailed Results Table** - One row per case with:
   - Case ID and description
   - Annual Heating (actual vs reference, pass/fail)
   - Annual Cooling (actual vs reference, pass/fail)
   - Peak Heating (actual vs reference, pass/fail)
   - Peak Cooling (actual vs reference, pass/fail)
   - Overall pass status
4. **Systematic Issues Section** - List recurring problems across cases
5. **Phase Progress** - Which phases complete, which requirements satisfied
6. **Appendix** - Raw data tables, test execution command

Create a design document or directly implement the template.

**Acceptance:**
- Report structure documented in this plan or implemented in code
- Stakeholder approval on content and format

### Task 2: Implement Report Generation Engine

**Priority:** P0
**Estimate:** 1 hour

Create `src/validation/reporter.rs` with:

```rust
pub struct ValidationReportGenerator {
    // Configuration
    output_path: PathBuf,
    template: Option<String>,
}

impl ValidationReportGenerator {
    pub fn new(output_path: PathBuf) -> Self;
    pub fn generate(&self, results: &[CaseValidationResult]) -> Result<(), Error>;
    pub fn render_markdown(&self, results: &[CaseValidationResult]) -> String;
}
```

Implementation:
- Accept an array of `CaseValidationResult` (or `ValidationReport` from validator)
- Compute aggregate statistics (pass rate, MAE, max deviation)
- Produce Markdown string with formatted tables
- Write to `docs/ASHRAE140_RESULTS.md` (or configured path)

**Key Functions:**
- `generate_full_report()` - Main entry point
- `render_summary_card()` - Top-level statistics
- `render_results_table()` - Detailed case table
- `render_systematic_issues()` - Issue summary
- `compute_metrics()` - Calculate MAE, pass rate, etc.

**Acceptance:**
- Compiles without errors
- Generates syntactically valid Markdown
- Report includes all required sections

### Task 3: Integrate with Validation Workflow

**Priority:** P1
**Estimate:** 30 minutes

Decide how report generation is triggered. Options:

**Option A: Post-Validation Hook**
Add `generate_report()` call after full validation suite completes in `tests/ashrae_140_validation.rs`.

**Option B: Standalone Command**
Create `src/bin/report.rs` that reads cached validation results or re-runs validation.

**Option C: CI/CD Integration**
Add GitHub Actions step that runs after tests and generates report.

**Recommendation:** Option A (simplest for now). Add to `tests/ashrae_140_validation.rs`:

```rust
#[test]
fn generate_validation_report() {
    let results = run_all_validations();
    let generator = ValidationReportGenerator::new("docs/ASHRAE140_RESULTS.md".into());
    generator.generate(&results).unwrap();
}
```

**Acceptance:**
- Running validation tests produces updated `docs/ASHRAE140_RESULTS.md`
- Report reflects current test results
- Report is committed to repository

### Task 4: Populate Systematic Issues Taxonomy

**Priority:** P2
**Estimate:** 20 minutes

Define common failure modes and categorize them:

- **BASE-04**: Denver TMY weather data issue
- **SOLAR-03**: Peak cooling under-prediction
- **MULTI-01**: Inter-zone radiation over-prediction (issue #273)
- **GROUND-01**: Ground coupling error

The report should attribute each failed metric to a known issue or mark as "unknown".

**Acceptance:**
- Systematic issues section meaningful
- Links to GitHub issues where applicable

# Plan 05-03: Hourly CSV Export System

## Objective

Implement a standalone command-line tool to export hourly simulation time series data to CSV format. This enables:
- External analysis in Python (pandas), R, or Excel
- Custom plots and visualizations beyond built-in reports
- Data sharing with collaborators who don't use Rust
- Archival of raw results for reproducibility

This addresses requirement REPORT-03.

## Background

The diagnostics module (Plan 05-02) will collect hourly data in memory. This plan provides a mechanism to export that data to portable CSV files. CSV is universally supported and ideal for post-processing.

## Locked Decisions

1. **Separate Binary:** Create `src/bin/export_csv.rs` as standalone CLI tool (not part of core library). Avoids forcing CSV dependency on all users.
2. **Export Structure:**
   ```
   output/csv/{case_id}/
   ├── case_900_zone0.csv
   ├── case_900_zone1.csv
   ├── case_960_zone0.csv
   ├── case_960_zone1.csv
   └── metadata.json
   ```
3. **CSV Columns:** hour, outdoor_temp, zone_temp, mass_temp, load_solar, load_internal, load_hvac, load_interzone, load_infiltration, cumulative_heating_kwh, cumulative_cooling_kwh
4. **Delimiter:** Comma (default), configurable to semicolon via `--delimiter` flag
5. ** invocation:** `cargo run --bin export_csv -- case_960` or `fluxion-export-csv --cases 900,960`

### Task 2: Create CLI Binary

**Priority:** P0
**Estimate:** 30 minutes

Create `src/bin/export_csv.rs`:

```rust
use fluxion::validation::{ashrae_140_cases::ASHRAE140Case, exporter::CsvExporter};
use fluxion::validation::ashrae_140_validator::ASHRAE140Validator;

fn main() -> Result<(), Error> {
    let args: Args = structopt::parse();
    let validator = ASHRAE140Validator::new();

    let cases = args.cases.as_deref().unwrap_or(&["900", "960"][..]);

    for case_str in cases {
        let case = match case_str {
            "900" => ASHRAE140Case::Case900,
            "960" => ASHRAE140Case::Case960,
            // ... all supported cases
            _ => panic!("Unknown case: {}", case_str),
        };

        let (report, Some(diags)) = validator.validate_case_with_diagnostics(case, true) else {
            eprintln!("Diagnostics not available for case {}", case_str);
            continue;
        };

        let exporter = CsvExporter::new(args.output_dir.clone(), args.delimiter);
        exporter.export_diagnostics(case_str, &diags, &case.spec())?;
        exporter.export_metadata(case_str, &case.spec(), &report)?;

        println!("Exported {} to {}/", case_str, args.output_dir.display());
    }

    Ok(())
}
```

Use `structopt` or `clap` for argument parsing (if available), otherwise simple env vars.

**Acceptance:**
- Binary builds: `cargo build --bin export_csv`
- Running `cargo run --bin export_csv -- --cases 900` produces CSV files
- CLI help accessible: `cargo run --bin export_csv -- --help`

       ```rust
       // Beam-to-mass distribution: 70% to thermal mass, 30% to interior surface
       let solar_beam_to_mass_fraction = 0.7;
       let phi_st_solar = &self.solar_gains * (1.0 - solar_beam_to_mass_fraction); // 30% to interior surface
       let phi_m_solar = &self.solar_gains * solar_beam_to_mass_fraction; // 70% to thermal mass
       ```

    3. Split mass gains to exterior and interior surfaces:
       ```rust
       // Split mass gains: 70% to exterior mass surface, 30% to interior mass surface
       let phi_m_env_solar = phi_m_solar * 0.7;
       let phi_m_int_solar = phi_m_solar * 0.3;
       ```

    4. Wire solar gains into the 5R1C thermal network energy balance equation:
       ```rust
       // Energy balance with solar contribution - CRITICAL WIRING
       // phi_i = (phi_i_total + phi_si + phi_mi + phi_m_int_solar) * T_a + phi_m_env_solar * T_m
       let phi_i = (&phi_i_total + &phi_si + &phi_mi + &phi_m_int_solar)
           * &T_a
           + &phi_m_env_solar * &T_m;

       // KEY WIRING: phi_m_env_solar * T_m is the term that integrates solar gains into thermal network
       // This term adds solar energy stored in thermal mass (phi_m_env_solar) multiplied by mass temperature (T_m)
       // to overall energy balance equation (phi_i)
       // Without this term, solar gains are calculated but never affect building temperature
       ```

    5. Ensure all solar gain variables are VectorField types to maintain CTA compatibility:
       ```rust

       ```

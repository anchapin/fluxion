<objective>
Fix peak load tracking to use actual HVAC demand instead of steady-state heat loss approximation, correcting peak cooling/heating load under-prediction.

Purpose: After fixing annual cooling energy in Plan 03-02, peak loads remain under-predicted (66-79% for cooling, 26-55% for heating). Root cause: peak load tracking uses steady-state heat loss (ss_heat_loss = h_ext * (setpoint - outdoor_temp)) which doesn't account for thermal mass buffering. The fix: use hvac_output_raw (actual HVAC demand calculated from free-floating temperature) which already includes thermal mass effects.

Output: Peak load tracking uses hvac_output_raw directly (no steady-state approximation), resulting in Case 900 peak cooling within [2.10, 3.50] kW and peak heating within [1.10, 2.10] kW.

    2. Identify issues with current peak load tracking:
       - **Issue 1: Steady-state heat loss approximation**
         - Current method: ss_heat_loss = h_ext * (setpoint - outdoor_temp)
         - This assumes building reaches steady-state instantly
         - For high-mass buildings, thermal mass buffers temperature swings, reducing peak loads
         - The thermal_mass_correction_factor is applied, but steady-state approximation is fundamentally incorrect

       - **Issue 2: Peak correction factor insufficient**
         - peak_thermal_mass_correction_factor is applied to steady-state heat loss
         - This is a multiplicative correction that doesn't capture thermal mass buffering physics
         - The correction factor may be too low, resulting in under-prediction

       - **Issue 3: Free-floating temperature not used**
         - Peak tracking uses outdoor temp: ss_heat_loss = h_ext * (setpoint - outdoor_temp)
         - Should use free-floating temp (t_i_free) which includes thermal mass buffering
         - Free-floating temp represents actual building response to outdoor conditions

       - **Issue 4: hvac_output_raw already calculated correctly**
         - hvac_output_raw = self.hvac_power_demand(hour_of_day_idx, &t_i_free, &sensitivity_val) (line 1885)
         - This already includes thermal mass effects via t_i_free calculation
         - Peak tracking should use hvac_output_raw directly, not recalculate with steady-state approximation

         a. Create a reqwest client.
         b. Perform HTTP GET to `remote_url`.
         c. If response status is not 200, return `anyhow::bail!` with HTTP status.
         d. Parse response body as JSON into `MultiReferenceDB` using `serde_json::from_str`.
         e. Validate the structure: ensure `version` and `source` are non-empty, at least one case exists, and for a sample case the metric hashes contain expected program keys (e.g., "EnergyPlus"). If validation fails, return `anyhow::bail!` with explanation.

            - Backup existing file to `docs/ashrae_140_references.json.bak`.
            - Write new JSON to `docs/ashrae_140_references.json` with pretty formatting (4-space indent).
            - Print: "Updated reference data to version {version} from {source}. Cases: {N}."
         g. If local file does not exist, simply write the fetched JSON.
       - Wrap all steps in `Result<()>` and propagate errors with context (`anyhow::Context`).
       - Add unit tests using `mockito` or `wiremock` to mock HTTP responses: test successful fetch and update, test invalid JSON, test schema validation failures, test network error handling.

    3. Ensure the module imports `MultiReferenceDB` and `reqwest`.

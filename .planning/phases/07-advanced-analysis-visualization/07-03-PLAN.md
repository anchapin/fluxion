    2. Implement `pub fn interpret_swing_metrics(metrics: &SwingMetrics, total_hours: usize) -> SwingInterpretation`:
       - **Thermal mass effectiveness**: If swing_range is small (< 5°C) and comfort_hours_pct is high (> 80%), classify as "High" (effective thermal mass stabilizes temperatures). If swing_range moderate (5-10°C) and comfort_hours_pct moderate (50-80%), classify as "Moderate". If swing_range large (> 10°) and comfort_hours_pct low (< 50%), classify as "Low" (insufficient thermal mass or extreme climate).
       - **Passive cooling potential**: If comfort_hours_pct >= 70% and avg_temp within comfort band (18-26°C), classify as "High" (building stays comfortable without cooling). If comfort_hours_pct 40-70% with avg_temp slightly above band (26-28°C), classify as "Moderate" (some cooling needed). If comfort_hours_pct < 40% or avg_temp > 28°C, classify as "Low" (significant cooling required).
       - **Passive heating potential**: If comfort_hours_pct >= 70% and avg_temp >= 18°C, classify as "High" (passive heating sufficient). If comfort_hours_pct 40-70% with avg_temp 15-18°C, classify as "Moderate". If avg_temp < 15°C or comfort_hours_pct < 40%, classify as "Low".
       - **Recommendations**: Generate a list of actionable suggestions:
         * If thermal mass effectiveness is "Low" and swing_range large: recommend increasing thermal mass or improving insulation.
         * If passive cooling potential is "Low" and comfort_hours_pct low due to high temps: recommend shading, reflective surfaces, increased ventilation, or smaller window-to-wall ratio.
         * If passive heating potential is "Low" and avg_temp low: recommend increased solar gain (larger windows on south facade), higher insulation, or reduced infiltration.
       - Consider both swing metrics and seasonal patterns (not available here, so simpler rules).
    3. Implement `pub fn generate_swing_report(interpretations: &[SwingInterpretation]) -> String` returning a Markdown-formatted report with a table summarizing cases and their classifications, followed by recommendation bullets.
    4. Write unit tests:
       - `test_interpretation_high_effectiveness`: metrics with swing_range=4.0, comfort_pct=90% → thermal_mass_effectiveness="High".
       - `test_interpretation_low_cooling`: metrics with avg_temp=29.0, comfort_pct=30% → passive_cooling_potential="Low".
       - `test_recommendations_generated`: ensure recommendations list is non-empty for low-performing cases.
    5. Ensure the SwingInterpretation and report function are exported and documented.
  </action>

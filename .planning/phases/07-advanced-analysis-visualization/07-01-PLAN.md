<task type="auto">
  <name>Task 3: Implement sensitivity batch evaluation, metrics, and CSV export</name>
  <files>
    - src/analysis/sensitivity.rs
  </files>
  <action>
    Extend src/analysis/sensitivity.rs:
    1. Define `pub struct SensitivityReport { pub parameters: Vec<String>, pub metrics: Vec<MetricSet> }` and `pub struct MetricSet { pub normalized_coeff: f64, pub cvrmse: f64, pub nmbe: f64, pub slope: f64 }`.
    2. Implement `pub fn run_sensitivity(design: &[Vec<f64>], case_builder: &CaseBuilder, use_surrogates: bool) -> Vec<f64>`:
       - Construct a `BatchOracle` from `case_builder` (or use builder to create a model then batch evaluate). Use `BatchOracle::evaluate_population(design.to_vec(), use_surrogates)` to get fitness scores.
    3. Implement `pub fn compute_metrics(design: &[Vec<f64>], outputs: &[f64]) -> SensitivityReport`:
       - For each parameter column i: extract the x-values and corresponding outputs.
       - Compute normalized sensitivity coefficient: (max(outputs) - min(outputs)) / baseline_output * 100%, where baseline_output = output at median parameter value or first row? For OAT, baseline is the first row (all at midpoint). For Sobol, use average output as baseline or define differently. We'll define: coefficient = (max - min) / (mean(outputs) + epsilon).
       - Perform linear regression (y = a + b*x) using `linregress` crate to get slope, R-squared.
       - Compute CVRMSE: RMSE / mean(y) * 100%, where RMSE = sqrt(mean((y_i - y_fit)^2)).
       - Compute NMBE: sum(y_i - y_fit) / (n * mean(y)) * 100%.
       - Store results and rank by absolute normalized coefficient.
    4. Implement `pub fn export_to_csv(report: &SensitivityReport, path: &Path) -> Result<(), Box<dyn std::error::Error>>` using the `csv` crate. Write header: `Rank,Parameter,NormalizedCoeff,CVRMSE,NMBE,Slope`. Write one row per parameter in rank order (starting at 1).
    5. Write unit tests:
       - `test_metrics_computation`: create synthetic data where one parameter has perfect linear relationship (y = 2*x + noise), assert slope ≈ 2, CVRMSE < 5%, etc.
       - `test_csv_export`: create a SensitivityReport, call export_to_csv to a temp file, read back and verify headers and that rows are sorted by descending NormalizedCoeff.
    6. Ensure no panic on edge cases (constant parameter).
  </action>

    cargo test -- --test-threads=1 --show-output sensitivity::metrics
  </verify>
  <done>Sensitivity metrics computed correctly; CSV export works; tests validate.</done>

<decisions_made>

- **Enrichment pattern**: Rather than replace all add_result_simple calls, use post-processing enrichment (enrich_with_multi_reference) to add per-program status to existing results. This is cleaner and maintains backward compatibility.
- **Status rule**: Overall status = PASS if EnergyPlus passes, else WARN if any program passes, else FAIL (as specified in VERIFICATION.md).
- **Aggregated refs**: Use envelope (min of mins, max of maxes) for aggregated ref_min/ref_max.
- **Fallback**: For free-floating metrics and missing cases, keep per_program=None and use simple validation.

</decisions_made>

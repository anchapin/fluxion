# Research insights on integration methods
From 02-RESEARCH.md:
"Backward Euler solver for implicit thermal mass update: Solves Cm * (Tm_new - Tm_old) / dt = Q_net(Tm_new), rearranges to (Cm/dt + h_tr_em + h_tr_ms) * Tm_new = Cm/dt * Tm_old + h_tr_em * t_ext + h_tr_ms * t_s + phi_m"

"Crank-Nicolson solver for semi-implicit thermal mass update: 2nd-order accurate, A-stable, better for oscillatory systems. Uses average of old and new heat fluxes."

    Key components:
    - `ThermalIntegrationMethod` enum: ExplicitEuler, BackwardEuler, CrankNicolson
    - `backward_euler_update()` function: Solves implicit equation (Cm/dt + h_tr_em + h_tr_ms) * Tm_new = RHS
    - `crank_nicolson_update()` function: Solves semi-implicit equation using average flux

    Backward Euler implementation:
    ```rust
    fn backward_euler_update(
        tm_old: f64,
        dt: f64,
        cm: f64,
        h_tr_em: f64,
        h_tr_ms: f64,
        t_ext: f64,
        t_surface: f64,
        phi_m: f64,
    ) -> f64 {
        let denom = cm / dt + h_tr_em + h_tr_ms;
        let numer = cm / dt * tm_old + h_tr_em * t_ext + h_tr_ms * t_surface + phi_m;
        numer / denom
    }
    ```

    Crank-Nicolson implementation:
    ```rust
    fn crank_nicolson_update(
        tm_old: f64,
        dt: f64,
        cm: f64,
        h_tr_em: f64,
        h_tr_ms: f64,
        t_ext: f64,
        t_surface: f64,
        phi_m: f64,
    ) -> f64 {
        let q_old = h_tr_em * (t_ext - tm_old) + h_tr_ms * (t_surface - tm_old) + phi_m;
        let a = h_tr_em + h_tr_ms;
        let b = h_tr_em * t_ext + h_tr_ms * t_surface + phi_m;
        let denom = cm / dt + 0.5 * a;
        let numer = cm / dt * tm_old + 0.5 * q_old + 0.5 * b;
        numer / denom
    }
    ```

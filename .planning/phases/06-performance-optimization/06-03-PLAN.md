    Create new file `src/ai/shared_batch_service.rs`:
    - Define structs: `InferenceRequest { inputs: Vec<Vec<f64>>, response_tx: oneshot::Sender<Vec<Vec<f64>>> }`.
    - `SharedBatchInferenceService` holds an `Arc<SurrogateManager>`, an `Arc<SegQueue<InferenceRequest>>` (from `crossbeam::queue::SegQueue`), a `DynamicBatchConfig`, and a worker thread.
    - `new(surrogate: SurrogateManager, config: DynamicBatchConfig)` starts the worker thread. The worker loop:
      1. Collect a batch: pop requests from queue (try_pop) until batch size reaches `config.max_batch_size` or wait timeout `config.wait_ms`.
      2. If batch empty, continue.
      3. Flatten batch into a single large batch (concatenate all input vectors).
      4. Call `surrogate.predict_loads_batched(&flat_batch)`.
      5. Split results back into per-request chunks and send via oneshot channels.
    - `submit(&self, inputs: &[Vec<f64>]) -> oneshot::Receiver<Vec<Vec<f64>>>` creates a oneshot channel, pushes request to queue, returns receiver.
    - Ensure thread safety: queue is `Arc<SegQueue<...>>`, worker moves cloned queue.
    In `tests/test_shared_batch_service.rs`:
    - Create a mock `SurrogateManager` (use `SurrogateManager::new()` which returns constant 1.2).
    - Create `SharedBatchInferenceService` with small batch config (max_batch_size=4, wait_ms=10).
    - Spawn multiple threads that submit requests concurrently (e.g., 10 threads each with input of [[20.0]]).
    - Verify that all receivers get the expected mock results.
    - Test that batched inference actually calls `predict_loads_batched` by counting calls (may need a mock wrapper).
  </action>

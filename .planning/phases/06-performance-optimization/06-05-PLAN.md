<task type="auto" tdd="true">
  <name>Task 2: Create GuardrailChecker and integrate into CLI</name>
  <files>
    src/validation/guardrails.rs
    src/bin/fluxion.rs
    tests/test_guardrail_exit_codes.rs
  </files>
  <action>
    - Create `src/validation/guardrails.rs`:
      * Define `pub struct GuardrailBaseline { pub mae: f64, pub max_deviation: f64, pub pass_rate: f64, pub validation_time_seconds: f64 }`.
      * Load function: `pub fn load_baseline(path: &str) -> Result<GuardrailBaseline, String>` that deserializes JSON from `docs/performance_baseline.json`. Use `serde_json`.
      * Check function: `pub fn check(report: &BenchmarkReport, baseline: &GuardrailBaseline) -> (bool, Vec<String>)` returns `(exit_success, list_of_failures)`. Logic:
        - if report.mae() > baseline.mae * 1.02 → failure
        - if report.max_deviation() > baseline.max_deviation * 1.10 → failure
        - if report.pass_rate() < baseline.pass_rate - 5.0 → failure
        - if report.duration_seconds() > baseline.validation_time_seconds * 1.10 → warning only (print but not failure)
      * Return exit success true only if no failures.
    - In `src/bin/fluxion.rs`:
      * After the `validate` command completes (after generating report and before exit), if `--ci` flag present or `env::var("CI") == Ok("true")`:
        - Call `BenchmarkReport::append_history()` (from Task 1) to log metrics.
        - Load baseline via `GuardrailBaseline::load("docs/performance_baseline.json")`. If file missing, print warning and skip guardrails (exit 0).
        - Call `GuardrailChecker::check(&report, &baseline)`. If failures exist, print each error and set exit code = 1. Otherwise, if time warning, print warning but exit 0.
      * For non-CI runs, still call `append_history()` but no guardrail checks.
    - In `tests/test_guardrail_exit_codes.rs`:
      * Unit tests for `GuardrailChecker::check` covering each threshold condition.
      * Construct synthetic BenchmarkReport with controllable metrics (using direct struct construction or helper) and baseline values.
      * Verify that exactly when rules trigger, the function returns (false, ...) indicating failure.
  </action>
  <verify>
    `cargo test --test test_guardrail_exit_codes` must pass, covering all guardrail scenarios.
  </verify>
  <done>Guardrail system implemented with CI integration and unit tests.</done>
</task>
